# 项目结构与使用说明

## 1. 目录结构说明

```
├── main.py                        # 主程序入口，命令行参数控制全流程
├── README.md                      # 项目说明文档（简明总览）
├── requirements.txt               # 依赖包列表
├── driver/                        # 浏览器驱动（如msedgedriver.exe）
├── data/
│   ├── preprocessed/              # 预处理后数据（csv等，供分析用）
│   ├── preprocessing/             # 原始及中间数据（如原始csv、临时文件）
│   ├── reporter/
│   │   ├── music_rank/            # 音乐榜单分析图片与报告
│   │   └── video_danmaku/         # 弹幕分析图片与报告
│   ├── raw/                       # 原始爬取数据（json等）
│   └── cookies/                   # 登录cookies存储
├── src/
│   ├── analyzer/                  # 分析与可视化模块（music_rank、video_danmaku分析、Notebook报告等）
│   ├── crawler/                   # 爬虫与登录模块
│   ├── preprocessing/             # 数据预处理模块
│   └── utils/                     # 工具函数、日志等
├── logs/                          # 日志文件
└── doc/                           # 项目文档、说明、分区说明等
```

- **driver/**：存放 EdgeDriver 浏览器驱动，需手动下载放入。
- **data/preprocessed/**：所有分析用的预处理csv数据。
- **data/reporter/**：所有分析报告生成的图片、md、Notebook等。
- **data/cookies/**：自动登录后保存的cookies。
- **src/analyzer/**：分析主逻辑、可视化、Jupyter报告。
- **logs/**：自动生成的运行日志，便于排查问题。
- **doc/**：本项目详细文档、分区说明、使用指引等。

## 2. 依赖环境与准备

- Python 3.8+
- 依赖包见 requirements.txt，安装命令：
  ```bash
  pip install -r requirements.txt
  ```
- EdgeDriver 下载：https://developer.microsoft.com/en-us/microsoft-edge/tools/webdriver/ ，放入 driver/ 目录

## 3. 典型使用流程

- 音乐榜单分析：
  ```bash
  python main.py --type music_rank --crawl --preprocess --analyze
  ```
- 视频弹幕分析（需指定视频id，支持BVID或AID）：
  ```bash
  python main.py --type video_danmaku --id BVxxxxxxxxx --crawl --preprocess --analyze
  ```
- 支持单步运行：`--login`、`--crawl`、`--preprocess`、`--analyze` 可任意组合
- analyze 阶段自动生成并打开对应 Jupyter 分析报告，并自动运行全部代码块

## 3.1 命令行参数详细说明

| 参数         | 是否必选 | 说明                                                         | 可选值/格式                | 示例                                                         |
|--------------|----------|--------------------------------------------------------------|----------------------------|--------------------------------------------------------------|
| --type       | 是       | 指定分析类型                                                 | music_rank, video_danmaku, login | --type music_rank                                            |
| --id         | 否       | 视频弹幕分析时需指定视频id（BVID或AID），音乐榜单无需此参数 | BVxxxxxxxxx 或 纯数字AID   | --id BV1xx411c7mD  或  --id 123456789                        |
| --crawl      | 否       | 执行数据采集                                                 | 无                         | --crawl                                                      |
| --preprocess | 否       | 执行数据预处理                                               | 无                         | --preprocess                                                 |
| --analyze    | 否       | 执行分析并生成可视化报告                                     | 无                         | --analyze                                                    |
| --login      | 否       | 执行B站登录（仅`--type login`时有效）                        | 无                         | --login                                                      |

- 参数可任意组合，未指定的阶段将跳过。
- 典型用法：
  - 一键全流程：`python main.py --type music_rank --crawl --preprocess --analyze`
  - 只分析弹幕：`python main.py --type video_danmaku --id BVxxxxxxxxx --analyze`
  - 只登录：`python main.py --type login`

## 3.2 分析报告自动生成路径与查看方式

- analyze 阶段会自动生成并打开对应的 Jupyter Notebook 分析报告，并自动运行全部代码块，所有图片自动嵌入。
- 报告及图片输出路径：
  - 音乐榜单分析报告：`src/analyzer/music_rank_analysis_report.ipynb`
  - 音乐榜单分析图片/markdown：`data/reporter/music_rank/`
  - 视频弹幕分析报告：`src/analyzer/video_danmaku_analysis_report.ipynb`
  - 视频弹幕分析图片/markdown：`data/reporter/video_danmaku/`
- 查看方式：
  - analyze 阶段结束后会自动打开 Notebook，可在 Jupyter Lab/Notebook 或 VSCode 直接查看。
  - 所有图片均以 markdown 方式嵌入报告，结论部分支持一键导出 markdown。

## 4. 常见问题与注意事项

- 首次运行建议先安装依赖并检查 Python 版本
- 若 Notebook 图表中文乱码，请确保本地已安装 SimHei 或 Microsoft YaHei 字体，并在分析报告开头设置：
  ```python
  import matplotlib
  matplotlib.rcParams['font.sans-serif'] = ['SimHei', 'Microsoft YaHei', 'Arial Unicode MS']
  matplotlib.rcParams['axes.unicode_minus'] = False
  ```
- 日志文件位于 logs/，便于排查问题
- 若需自定义分析或可视化，可直接修改 src/analyzer/ 下的 Notebook 或 Python 脚本

## 5. 参考与扩展

- 支持自定义扩展分析、可视化、数据融合等高级功能。

## 6. 开发者扩展建议

### 6.1 新增分析类型
- 在 `src/analyzer/` 下新建分析脚本和 Notebook，按现有 music_rank/video_danmaku 结构实现。
- 在 `main.py` 增加 `--type` 分支，调用新分析流程。

### 6.2 特征工程与可视化扩展
- 在对应分析脚本中添加特征工程、聚类、情感分析、可视化等函数。
- 推荐将图片统一输出到 `data/reporter/` 下对应子目录，便于报告自动引用。
- Notebook 可按章节分节，便于结构化输出和自动生成 markdown。

### 6.3 复用与接口说明
- 公共工具函数可放在 `src/utils/`，如日志、文件查找、图片保存等。
- 数据预处理建议输出标准 csv，便于后续分析复用。
- 可参考现有 analyzer 代码风格，支持自动查找最新数据、自动生成报告。
